# branchdiff — Guidelines

Three audiences, three docs:

| Doc | For whom | What's inside |
|---|---|---|
| [USAGE.md](./USAGE.md) | **End users** of the `branchdiff` npm CLI | Install, first run, commands, flags, troubleshooting |
| [AI-REVIEW.md](./AI-REVIEW.md) | **Users driving an AI assistant** (Claude Code, Cursor, Codex, Copilot, Gemini, …) | Prompt templates for review / resolve / tour / summary — no plugin required |
| [DEVELOPMENT.md](./DEVELOPMENT.md) | **Contributors** hacking on the repo | Clone, build, local-link, testing, releasing |

Start with [USAGE.md](./USAGE.md) if you just want to use branchdiff.
Start with [AI-REVIEW.md](./AI-REVIEW.md) if you want an AI to review / fix / walk through code.
Start with [DEVELOPMENT.md](./DEVELOPMENT.md) if you want to work on branchdiff itself.
